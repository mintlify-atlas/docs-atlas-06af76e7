Get up and running with STRIDE GPT to generate your first threat model using AI-powered security analysis.

## Prerequisites

Before you begin, you'll need:
- An API key from at least one AI provider (OpenAI, Anthropic, Google, Mistral, or Groq)
- OR a local model running via [Ollama](https://ollama.ai) or LM Studio

<Info>
Don't have an API key? Check out the [installation configuration guide](/installation/configuration) for links to get started with each provider.
</Info>

## Launch the Application

<Steps>
  <Step title="Start STRIDE GPT">
    After installation, launch the application:
    
    <CodeGroup>
    ```bash Local
    streamlit run main.py
    ```
    
    ```bash Docker
    docker run -p 8501:8501 --env-file .env mrwadams/stridegpt
    ```
    </CodeGroup>
    
    The application will open in your browser at `http://localhost:8501`
  </Step>
  
  <Step title="Configure Your AI Provider">
    In the sidebar, select your preferred model provider and enter your API key:
    
    ```python Example: OpenAI Configuration
    1. Select "OpenAI API" from the dropdown
    2. Enter your OpenAI API key
    3. Choose a model (e.g., gpt-5.2)
    ```
    
    <Note>
    Your API keys are stored only in your browser session and are never saved to disk.
    </Note>
  </Step>
  
  <Step title="Describe Your Application">
    In the main area, provide details about your application:
    
    ```text Example Description
    A web application that allows users to create, store, and share 
    personal notes. The application is built using React frontend and 
    Node.js backend with MongoDB database. Users authenticate via 
    OAuth2 with Google or Facebook. Notes are encrypted at rest and 
    only accessible by the user who created them. The application 
    supports real-time collaboration with other users.
    ```
    
    Fill in additional details:
    - **Application Type**: Web application, Mobile app, API, etc.
    - **Authentication**: OAuth, JWT, Session-based, etc.
    - **Internet Facing**: Yes/No
    - **Sensitive Data**: PII, Financial, Health records, etc.
  </Step>
  
  <Step title="Generate Threat Model">
    Click **"Generate Threat Model"** and wait for the AI to analyze your application.
    
    The threat model will display threats categorized by STRIDE methodology:
    - **S**poofing
    - **T**ampering
    - **R**epudiation
    - **I**nformation Disclosure
    - **D**enial of Service
    - **E**levation of Privilege
    
    <Warning>
    Generation time varies based on your chosen model and API rate limits. 
    Reasoning models (GPT-5 series) take longer but provide deeper analysis.
    </Warning>
  </Step>
  
  <Step title="Review Your Results">
    Examine the generated threats in the table format:
    
    | Threat Type | Scenario | Potential Impact |
    |-------------|----------|------------------|
    | Spoofing | Attacker impersonates OAuth provider... | Unauthorized account access... |
    | Tampering | Man-in-the-middle attack on API calls... | Data modification... |
    
    Review the **Improvement Suggestions** to enhance your threat model in the next iteration.
    
    Download your results as Markdown using the **Download** button.
  </Step>
</Steps>

## Optional: Advanced Features

### Upload Architecture Diagrams

For supported models (GPT-4+, Claude, Gemini), upload architecture diagrams to automatically generate application descriptions:

```python
1. Click "Upload architecture diagram"
2. Select your diagram (JPG, PNG)
3. Wait for AI analysis
4. Review and edit the generated description
```

### Analyze GitHub Repositories

Automatically analyze codebases from GitHub:

```python
1. Enter your GitHub API key in the sidebar
2. Paste the repository URL (e.g., https://github.com/owner/repo)
3. STRIDE GPT will analyze README and code files
4. Generated description appears in the text area
```

<Info>
GitHub analysis supports both public GitHub and GitHub Enterprise instances.
</Info>

### Use Local Models

For air-gapped or privacy-sensitive environments:

```python
1. Select "Ollama" or "LM Studio Server" as provider
2. Configure your local endpoint
3. Select from available local models
4. Generate threat models entirely offline
```

Learn more in the [Ollama](/providers/ollama) and [LM Studio](/providers/lm-studio) guides.

## Security Best Practices

<Warning>
Always follow security best practices when using STRIDE GPT:
- Store API keys in `.env` files, not in code
- Set spending limits in your AI provider dashboard
- Review generated outputs—AI models can make mistakes
- Avoid sharing real credentials in application descriptions
</Warning>

For comprehensive security guidance, see [Security Best Practices](/security/best-practices).
