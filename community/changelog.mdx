## Version 0.15 (Latest)

<Note>
Version 0.15 introduces comprehensive support for threat modeling agentic AI and generative AI applications with OWASP integration.
</Note>

### Agentic AI Application Support

Added comprehensive support for threat modeling agentic AI systems with OWASP Top 10 for Agentic Applications (ASI01-ASI10) integration:

- Select "Agentic AI application" to automatically apply relevant threat categories
- LLM automatically detects architectural patterns from system descriptions
- Each ASI risk is mapped to the appropriate STRIDE category for consistent analysis

### Generative AI Application Support

Added support for threat modeling GenAI applications:

- OWASP LLM Top 10 (LLM01-LLM10) integration
- Select "Generative AI application" for tailored LLM-specific threat analysis
- Agentic AI applications include both LLM and ASI risk categories for comprehensive coverage

### MAESTRO-Inspired Architectural Pattern Detection

Enhanced agentic threat analysis with automatic detection of architectural patterns, inspired by the [Cloud Security Alliance MAESTRO framework](https://cloudsecurityalliance.org/blog/2025/02/06/agentic-ai-threat-modeling-framework-maestro):

<Accordion title="RAG/Retrieval Systems">
- Vector store poisoning
- Embedding manipulation
- Cross-tenant leakage
</Accordion>

<Accordion title="Multi-Agent Systems">
- Agent impersonation
- Inter-agent message tampering
- Cascading compromises
</Accordion>

<Accordion title="Code Execution Environments">
- Sandbox escape
- Container breakout
- Malicious code generation
</Accordion>

<Accordion title="Tool/MCP Ecosystems">
- Rogue tool providers
- Supply chain attacks
- Confused deputy vulnerabilities
</Accordion>

<Accordion title="Persistent Memory">
- Memory poisoning
- Cross-session leakage
- State manipulation
</Accordion>

<Accordion title="Autonomous Operations">
- Rogue agent persistence
- Human oversight bypass
- Runaway loops
</Accordion>

### Cross-Layer Threat Analysis

Threat models now include cross-component attack chains showing how compromise in one architectural layer can enable attacks on others (e.g., RAG poisoning → agent goal hijack → tool misuse → data exfiltration).

### Model Updates

- **OpenAI**: Updated to GPT-5.2 series models (gpt-5.2, gpt-5.2-pro, gpt-5-mini, gpt-5-nano, gpt-5, gpt-4.1). Removed deprecated models.
- **Anthropic**: Updated to Claude 4.5 series (claude-sonnet-4-5, claude-haiku-4-5, claude-opus-4-5). Added "Enable Extended Thinking" checkbox. Increased max tokens to 32k (standard) and 48k (thinking mode).
- **Google**: Added Gemini 3 preview models (gemini-3-pro-preview, gemini-3-flash-preview) alongside Gemini 2.5 models. Extended thinking mode support.
- **Mistral**: Updated to latest versioned models including Mistral Large 3, Medium 3.1, Small 3.2, Ministral 3 (8B/14B), and Magistral 1.2 (Medium/Small).

### Other Improvements

- New project logo with cleaner, minimal shield + neural network design
- Enhanced improvement suggestions for agentic applications (prompts for framework details, communication protocols, sandbox mechanisms)
- Bug fix: Resolved issue where threat model output would disappear after clicking download button
