## Model Naming Convention

Mistral uses a versioned naming scheme:

- **mistral/ministral/magistral**: Model family
- **large/medium/small**: Size/capability tier
- **YYMM**: Release year and month (25 = 2025)

**Examples**:
- `mistral-large-2512` = Large model from December 2025
- `magistral-medium-2509` = Medium reasoning model from September 2025

## Setup

<Steps>
  <Step title="Get API Key">
    1. Visit the [Mistral Console](https://console.mistral.ai/api-keys/)
    2. Sign in or create an account
    3. Navigate to **API Keys** section
    4. Click **Create new key**
    5. Copy the generated key
    
    <Warning>
    Store your API key securely. It provides access to your Mistral account.
    </Warning>
  </Step>
  
  <Step title="Configure in STRIDE GPT">
    In the STRIDE GPT sidebar:
    
    1. Select **"Mistral API"** from model provider dropdown
    2. Paste your API key in the password field
    3. Choose your preferred Mistral model
  </Step>
  
  <Step title="Environment Variable (Optional)">
    Set as environment variable:
    
    ```bash
    export MISTRAL_API_KEY="your-key-here"
    ```
    
    Or in `.env` file:
    ```
    MISTRAL_API_KEY=your-key-here
    ```
  </Step>
</Steps>

## Best Practices

### Choosing the Right Model

<Steps>
  <Step title="Start with Mistral Medium 2508">
    Use as your default for most applications:
    - Good balance of cost and quality
    - 128K context window
    - Fast response times
    - Reliable results
  </Step>
  
  <Step title="Upgrade to Large for Complex Apps">
    Use Mistral Large 2512 when:
    - Application is security-critical
    - Complex architecture with many components
    - Need comprehensive OWASP coverage
    - Budget allows for premium model
  </Step>
  
  <Step title="Use Magistral for Reasoning">
    Use Magistral models when:
    - Analyzing complex agentic AI systems
    - Need deep reasoning about attack chains
    - Application has 40K tokens or less context
    - Reasoning depth more important than speed
  </Step>
  
  <Step title="Use Ministral for Speed/Cost">
    Use Ministral 8B or 14B when:
    - Rapid prototyping required
    - Budget constraints
    - Simple applications
    - Development phase analysis
  </Step>
</Steps>

## Troubleshooting

<Accordion title="API Key Invalid">
  **Symptom**: Authentication failure
  
  **Solutions**:
  - Verify key from Mistral Console
  - Check for extra spaces when pasting
  - Ensure key has appropriate permissions
  - Try regenerating the key
  - Check account status (payment, quotas)
</Accordion>

<Accordion title="Context Length Exceeded">
  **Symptom**: Error about maximum context length
  
  **Solutions**:
  - **For Mistral Small**: Switch to Medium or Large (128K context)
  - **For Magistral**: Switch to standard Mistral model
  - Reduce token limit in Advanced Settings
  - Simplify application description
  - Avoid GitHub analysis with Small/Magistral models
</Accordion>

<Accordion title="JSON Parsing Errors">
  **Symptom**: "Failed to parse JSON response"
  
  **Explanation**: Mistral sometimes includes markdown formatting
  
  **Solutions**:
  - STRIDE GPT automatically strips markdown blocks
  - Retry the request
  - Try a different model (Medium vs Large)
  - Report persistent issues
</Accordion>

<Accordion title="Poor Quality Results">
  **Symptom**: Threat model seems incomplete or generic
  
  **Solutions**:
  - Upgrade from Ministral 8B to 14B or standard models
  - Switch from Small to Medium or Large
  - Provide more detailed application description
  - Specify application type correctly (Agentic AI, GenAI, etc.)
  - Try Magistral for complex scenarios
</Accordion>

<Accordion title="Slow Response Times">
  **Symptom**: Longer than expected wait times
  
  **Explanation**: Magistral models use reasoning and are naturally slower
  
  **Solutions**:
  - Use Ministral 8B for fastest results
  - Switch from Magistral to standard Mistral
  - Use Mistral Small for simple apps
  - Check Mistral service status
</Accordion>
