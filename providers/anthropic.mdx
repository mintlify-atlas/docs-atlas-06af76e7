## Extended Thinking Mode

Claude's unique extended thinking mode provides enhanced reasoning for complex security analysis:

<Note>
**What is Extended Thinking?**

Extended thinking allocates additional tokens (16,000) for Claude to internally reason through the problem before responding. This enables:

- Deeper threat analysis across STRIDE categories
- Multi-step attack chain reasoning
- Cross-layer vulnerability identification
- More comprehensive OWASP LLM/ASI coverage

**Trade-offs**:
- ✅ More thorough and accurate threat models
- ✅ Better understanding of complex agentic architectures
- ❌ Longer response times (60-120 seconds)
- ❌ Higher token consumption and cost
</Note>

### Enabling Extended Thinking

In the STRIDE GPT sidebar:

1. Select **Anthropic API** as model provider
2. Choose any Claude 4.5 model
3. Check the **"Enable Extended Thinking"** checkbox

### When to Use Extended Thinking

<Tabs>
  <Tab title="Use Extended Thinking For">
    - **Agentic AI applications** with complex tool/MCP integrations
    - **Multi-agent systems** requiring deep architectural understanding
    - **Security-critical applications** where thoroughness is paramount
    - **Complex RAG pipelines** with multiple data sources
    - **Applications with custom LLM implementations**
    - When you need **maximum depth and precision**
  </Tab>
  
  <Tab title="Skip Extended Thinking For">
    - **Simple web applications** with standard architecture
    - **Rapid prototyping** and iterative refinement
    - **Cost-sensitive projects** with budget constraints
    - **Time-sensitive analysis** requiring quick results
    - **Traditional applications** without AI components
  </Tab>
</Tabs>

## Setup

<Steps>
  <Step title="Get API Key">
    1. Visit the [Anthropic Console](https://console.anthropic.com/settings/keys)
    2. Sign in or create an account
    3. Navigate to **API Keys** section
    4. Click **Create Key**
    5. Copy the generated key (starts with `sk-ant-`)
    
    <Warning>
    Store your API key securely. It cannot be recovered if lost.
    </Warning>
  </Step>
  
  <Step title="Configure in STRIDE GPT">
    In the STRIDE GPT sidebar:
    
    1. Select **"Anthropic API"** from model provider dropdown
    2. Paste your API key in the password field
    3. Choose your preferred Claude 4.5 model
    4. Optionally enable **Extended Thinking**
  </Step>
  
  <Step title="Environment Variable (Optional)">
    Set as environment variable:
    
    ```bash
    export ANTHROPIC_API_KEY="sk-ant-..."
    ```
    
    Or in `.env` file:
    ```
    ANTHROPIC_API_KEY=sk-ant-...
    ```
  </Step>
</Steps>

## Token Management

### Token Limits by Mode

| Mode | Max Tokens | Thinking Budget | Total Capacity |
|------|-----------|-----------------|----------------|
| **Standard** | 32,768 | 0 | 200,000 context |
| **Extended Thinking** | 48,000 | 16,000 | 200,000 context |

<Note>
**Context Window**: All Claude 4.5 models support up to 200,000 tokens of context, suitable for analyzing medium to large applications and codebases.
</Note>

### Optimizing Token Usage

For GitHub repository analysis:

1. **Adjust in Advanced Settings**: Set token limit between 4K - 200K
2. **Prioritize important files**: README, main application files are analyzed first
3. **Truncation handling**: STRIDE GPT automatically truncates if approaching limit

## Troubleshooting

<Accordion title="Invalid API Key Error">
  **Symptom**: "Invalid Anthropic API key"
  
  **Solutions**:
  - Verify key format: `sk-ant-api...`
  - Check key status in Anthropic Console
  - Ensure key hasn't been revoked
  - Try generating a new key
</Accordion>

<Accordion title="Timeout Errors with Extended Thinking">
  **Symptom**: Request times out after 10 minutes
  
  **Solutions**:
  - Disable extended thinking and retry
  - Simplify application description
  - Break complex app into smaller components
  - Use Haiku or Sonnet instead of Opus
  - Check Anthropic service status
</Accordion>

<Accordion title="JSON Parsing Failures">
  **Symptom**: "Failed to parse Claude response"
  
  **Explanation**: Sometimes Claude includes markdown formatting in JSON responses
  
  **Solutions**:
  - STRIDE GPT automatically strips markdown code blocks
  - Retry the request (usually works on second attempt)
  - Switch to standard mode if thinking mode fails consistently
</Accordion>

<Accordion title="Incomplete Threat Models">
  **Symptom**: Threat model seems less comprehensive than expected
  
  **Solutions**:
  - Enable **extended thinking** for more depth
  - Provide more detailed application description
  - Switch from Haiku to Sonnet or Opus
  - Explicitly mention application type (Agentic AI, GenAI, etc.)
</Accordion>

<Accordion title="Rate Limit Errors">
  **Symptom**: "Rate limit exceeded"
  
  **Solutions**:
  - Check usage tier in Anthropic Console
  - Wait for rate limit reset (typically 1 minute)
  - Upgrade to higher usage tier
  - Reduce concurrent requests
</Accordion>

## Advanced Configuration

### Timeout Settings

STRIDE GPT sets different timeouts based on mode:

```python
# Standard mode
timeout = 300  # 5 minutes

# Extended thinking mode
timeout = 600  # 10 minutes
```

### Token Allocation

```python
# Standard mode
max_tokens = 32768

# Extended thinking mode
max_tokens = 48000
thinking_budget = 16000
```

Claude uses specialized system prompts:

```
You are a JSON-generating assistant. You must ONLY output 
valid, parseable JSON with no additional text or formatting.
```

```
You are a cyber security expert. Use your extended reasoning 
capabilities to deeply analyze threats across all STRIDE 
categories and applicable OWASP risks.
```
