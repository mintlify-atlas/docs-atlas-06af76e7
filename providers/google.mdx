## Built-in Thinking Mode

Gemini 2.5 and 3 models include automatic thinking capabilities:

<Tabs>
  <Tab title="How It Works">
    Gemini models automatically use internal reasoning:
    
    1. **Analysis Phase**: Model examines application structure
    2. **Thinking Phase**: 1024 tokens allocated for internal reasoning
    3. **Synthesis Phase**: Generates comprehensive threat model
    
    This happens transparently without manual configuration.
  </Tab>
  
  <Tab title="Thinking Budget">
    ```python
    # Automatically configured for Gemini 2.5/3
    config = google_types.GenerateContentConfig(
        thinking_config=google_types.ThinkingConfig(
            thinking_budget=1024
        )
    )
    ```
  </Tab>
  
  <Tab title="Visibility">
    STRIDE GPT captures thinking content:
    
    - Stored in session state
    - Available for debugging
    - Shows reasoning process
    - Helps understand model decisions
  </Tab>
</Tabs>

<Info>
Unlike Anthropic's optional extended thinking, Gemini's thinking mode is always active and optimized for the model's architecture.
</Info>

## Setup

<Steps>
  <Step title="Generate API Key">
    1. Visit [Google AI Studio](https://makersuite.google.com/app/apikey)
    2. Sign in with your Google account
    3. Click **"Get API Key"** or **"Create API Key"**
    4. Copy the generated key
    
    <Warning>
    Keep your API key confidential. Do not commit to version control.
    </Warning>
  </Step>
  
  <Step title="Configure in STRIDE GPT">
    In the application sidebar:
    
    1. Select **"Google AI API"** from model provider dropdown
    2. Paste API key in the password field
    3. Choose your preferred Gemini model
  </Step>
  
  <Step title="Optional: Environment Variable">
    Set as environment variable:
    
    ```bash
    export GOOGLE_API_KEY="AIza..."
    ```
    
    Or in `.env` file:
    ```
    GOOGLE_API_KEY=AIza...
    ```
  </Step>
</Steps>

## Safety Settings

STRIDE GPT configures safety settings to allow security content:

```python
safety_settings = [
    SafetySetting(
        category=HarmCategory.HARM_CATEGORY_DANGEROUS_CONTENT,
        threshold=HarmBlockThreshold.BLOCK_NONE
    ),
    SafetySetting(
        category=HarmCategory.HARM_CATEGORY_HARASSMENT,
        threshold=HarmBlockThreshold.BLOCK_NONE
    ),
    # ... other categories
]
```

<Info>
**Why This Matters**: Threat modeling content involves discussing attack scenarios, vulnerabilities, and exploits. These safety settings prevent Gemini from refusing to analyze security threats.
</Info>

### Leveraging Large Context

<Tabs>
  <Tab title="For Large Repos">
    1. **Set high token limit**: Use 500K-1M in Advanced Settings
    2. **Include full context**: Don't truncate README or main files
    3. **Let Gemini analyze everything**: The model can handle it
    4. **Review token usage**: Check estimated tokens in analysis summary

  <Tab title="For Iterative Analysis">

## Troubleshooting

<Accordion title="API Key Authentication Failed">
  **Symptom**: "Invalid API key" or authentication error
  
  **Solutions**:
  - Verify key format starts with `AIza`
  - Check key hasn't been restricted in Google Cloud Console
  - Ensure Gemini API is enabled for your project
  - Try generating a new key
</Accordion>

<Accordion title="Safety Filter Blocking">
  **Symptom**: Response blocked due to safety filters
  
  **Explanation**: Despite BLOCK_NONE settings, some extreme scenarios may be blocked
  
  **Solutions**:
  - Rephrase threat scenarios less aggressively
  - Avoid explicit attack tool names
  - Focus on defensive perspective
  - Try a different model (2.5 vs 3)
</Accordion>

<Accordion title="Slow Response with Large Context">
  **Symptom**: Longer than expected response times
  
  **Explanation**: Even with 1M context, very large inputs take time to process
  
  **Solutions**:
  - Use Flash variants for speed
  - Consider splitting extremely large repositories
  - Reduce token limit if full analysis not needed
  - Try during off-peak hours
</Accordion>

<Accordion title="JSON Parsing Errors">
  **Symptom**: "Failed to parse JSON response"
  
  **Solutions**:
  - STRIDE GPT automatically cleans markdown code blocks
  - Retry the request (usually succeeds)
  - Try Gemini 2.5 instead of 3 (more stable formatting)
  - Report consistent issues for debugging
</Accordion>

<Accordion title="Missing OWASP Mappings">
  **Symptom**: Threat model lacks OWASP LLM/ASI codes
  
  **Solutions**:
  - Explicitly specify application type (Agentic AI, GenAI)
  - Mention LLM/agent components in description
  - Provide details about AI architecture
  - Try Gemini 3 Pro for better OWASP coverage
</Accordion>
