## Setup

<Steps>
  <Step title="Obtain API Key">
    Navigate to the [OpenAI Platform](https://platform.openai.com/account/api-keys) and generate an API key.
    
    <Warning>
    Keep your API key secure. Never commit it to version control or share it publicly.
    </Warning>
  </Step>
  
  <Step title="Configure in STRIDE GPT">
    In the STRIDE GPT sidebar:
    
    1. Select **"OpenAI API"** from the model provider dropdown
    2. Enter your API key in the password field
    3. Choose your preferred model from the dropdown
  </Step>
  
  <Step title="Optional: Environment Variable">
    Alternatively, set the API key as an environment variable:
    
    ```bash
    export OPENAI_API_KEY="sk-..."
    ```
    
    Or add to `.env` file:
    ```
    OPENAI_API_KEY=sk-...
    ```
  </Step>
</Steps>

## Reasoning Capabilities

GPT-5 models use extended internal reasoning before generating responses:

<Note>
**How GPT-5 Reasoning Works**

1. **Analysis Phase**: The model internally analyzes the application description, considering multiple threat vectors
2. **Reasoning Phase**: It explores various attack scenarios and maps them to STRIDE categories
3. **Synthesis Phase**: The model generates a comprehensive threat model with detailed scenarios

This process takes 30-90 seconds but produces significantly more thorough results than standard models.
</Note>

### Reasoning System Prompts

STRIDE GPT uses specialized system prompts for GPT-5 models:

```
You are a cyber security expert with 20+ years of experience.

Task: Generate a comprehensive STRIDE threat model

Approach:
1. Analyze application description and architecture
2. For each STRIDE category, identify specific threats
3. Consider LLM-specific risks (LLM01-LLM10) for GenAI apps
4. Consider agentic risks (ASI01-ASI10) for agentic apps
5. Provide detailed scenarios and impact analysis
```

### Adjusting Token Limits

<Warning>
When analyzing large GitHub repositories, monitor token usage to avoid exceeding context limits.
</Warning>

To adjust token limits:

1. Open **Advanced Settings** in the sidebar
2. Use the slider to set maximum token limit (4,000 - max for selected model)
3. STRIDE GPT will automatically truncate repository analysis if approaching the limit

### Token Estimation

STRIDE GPT uses `tiktoken` for accurate token counting:

```python
import tiktoken

enc = tiktoken.encoding_for_model("gpt-5.2")
tokens = enc.encode(text)
print(f"Token count: {len(tokens)}")
```

## Performance & Timing

### Expected Response Times

<Note>
Reasoning models take significantly longer than standard models due to internal thinking processes.
</Note>

| Model | Threat Model | Attack Tree | Mitigations | DREAD |
|-------|-------------|-------------|-------------|-------|
| gpt-5.2 | 60-90s | 45-60s | 30-45s | 30-40s |
| gpt-5.2-pro | 90-120s | 60-75s | 45-60s | 40-50s |
| gpt-5-mini | 30-45s | 25-35s | 20-30s | 20-25s |
| gpt-5-nano | 20-30s | 15-25s | 15-20s | 15-20s |
| gpt-4.1 | 15-25s | 12-20s | 10-15s | 10-15s |

### Why So Slow?

<Info>
**Rate Limits**: Free-tier API keys have strict rate limits. Upgrade to a paid account for faster responses.

**Reasoning Process**: GPT-5 models spend most of their time in internal reasoning, analyzing threats from multiple angles before responding.
</Info>

## Troubleshooting

### Common Issues

<Accordion title="API Key Invalid">
  **Error**: "Invalid API key"
  
  **Solutions**:
  - Verify key format starts with `sk-`
  - Check key hasn't expired in OpenAI dashboard
  - Ensure key has appropriate permissions
  - Try regenerating the API key
</Accordion>

<Accordion title="Rate Limit Exceeded">
  **Error**: "Rate limit reached"
  
  **Solutions**:
  - Upgrade to paid API account (Tier 1+)
  - Wait for rate limit window to reset (typically 1 minute)
  - Reduce frequency of requests
  - Use gpt-5-nano which has lower token consumption
</Accordion>

<Accordion title="Context Length Exceeded">
  **Error**: "Maximum context length exceeded"
  
  **Solutions**:
  - Reduce GitHub analysis token limit in Advanced Settings
  - Simplify application description
  - Switch to gpt-4.1 with 1M token context
  - Remove unnecessary code files from analysis
</Accordion>

<Accordion title="Slow Response Times">
  **Issue**: Model taking >2 minutes to respond
  
  **Explanations**:
  - Normal for GPT-5.2-pro on complex applications
  - Free-tier rate limiting causing delays
  - Server-side queuing during high demand
  
  **Solutions**:
  - Switch to gpt-5-mini or gpt-5-nano for faster results
  - Upgrade to paid API account
  - Retry during off-peak hours
</Accordion>

<Accordion title="Empty or Truncated Response">
  **Error**: "Empty response from model"
  
  **Solutions**:
  - Check OpenAI service status
  - Verify model availability in your region
  - Ensure sufficient API credits
  - Try a different model (gpt-4.1)
</Accordion>
