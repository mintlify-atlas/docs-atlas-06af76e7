## Setup

<Steps>
  <Step title="Get API Key">
    1. Visit the [Groq Console](https://console.groq.com/keys)
    2. Sign in or create an account
    3. Navigate to **API Keys** section
    4. Click **Create API Key**
    5. Copy the generated key (starts with `gsk_`)
    
    <Warning>
    Keep your Groq API key secure. It provides access to fast inference resources.
    </Warning>
  </Step>
  
  <Step title="Configure in STRIDE GPT">
    In the STRIDE GPT sidebar:
    
    1. Select **"Groq API"** from model provider dropdown
    2. Paste your API key in the password field
    3. Choose your preferred model
  </Step>
  
  <Step title="Environment Variable (Optional)">
    Set as environment variable:
    
    ```bash
    export GROQ_API_KEY="gsk_..."
    ```
    
    Or in `.env` file:
    ```
    GROQ_API_KEY=gsk_...
    ```
  </Step>
</Steps>

## Reasoning Model Support

DeepSeek R1 provides reasoning capabilities similar to GPT-5:

### How It Works

<Steps>
  <Step title="Internal Reasoning">
    DeepSeek R1 performs extended thinking before responding:
    ```
    <think>
    [Internal reasoning process]
    - Analyzing STRIDE categories
    - Considering attack vectors
    - Mapping to OWASP frameworks
    </think>
    ```
  </Step>
  
  <Step title="Response Extraction">
    STRIDE GPT automatically:
    - Detects `<think>` tags in output
    - Extracts reasoning content
    - Displays in expandable UI element
    - Parses final response separately
  </Step>
  
  <Step title="Visible Reasoning">
    Users can view the model's thought process:
    - Click "View model's reasoning process"
    - See step-by-step analysis
    - Understand threat identification logic
  </Step>
</Steps>

<Note>
Unlike OpenAI's GPT-5 or Anthropic's Extended Thinking, DeepSeek R1's reasoning is embedded in the output rather than being a separate API parameter.
</Note>

## Troubleshooting

<Accordion title="API Key Invalid">
  **Symptom**: Authentication failure
  
  **Solutions**:
  - Verify key format starts with `gsk_`
  - Check key status in Groq Console
  - Ensure key hasn't been revoked
  - Try regenerating the key
</Accordion>

<Accordion title="Rate Limit Exceeded">
  **Symptom**: "Rate limit reached"
  
  **Solutions**:
  - Groq has generous free tier but limits exist
  - Wait for rate limit reset (typically 1 minute)
  - Upgrade to paid tier for higher limits
  - Space out requests
</Accordion>

<Accordion title="Reasoning Tags in Output">
  **Symptom**: `<think>` tags visible in threat model
  
  **Explanation**: Normal for DeepSeek R1 - STRIDE GPT extracts these
  
  **Solutions**:
  - STRIDE GPT automatically handles this
  - Reasoning shown in expandable section
  - If not working, report as bug
</Accordion>

<Accordion title="Quality Lower Than Expected">
  **Symptom**: Threat model seems incomplete
  
  **Solutions**:
  - Upgrade from Llama 8B to 70B
  - Try GPT-OSS 120B for more depth
  - Use DeepSeek R1 for complex scenarios
  - Provide more detailed application description
</Accordion>

<Accordion title="JSON Parsing Failures">
  **Symptom**: "Failed to parse JSON response"
  
  **Solutions**:
  - STRIDE GPT has special handling for Groq responses
  - Retry the request
  - Try a different model
  - Report persistent issues
</Accordion>
